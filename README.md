# AI Customer Support Triage & Resolution System

## Overview
This project is a production-style AI system designed to improve e-commerce customer support operations by automatically understanding, prioritizing, routing, and assisting with customer support tickets.

The system integrates NLP-based intent classification, sentiment and urgency detection, SLA breach risk prediction, intelligent agent routing, and AI-assisted reply drafting with human-in-the-loop control.

## Business Problem
E-commerce support teams face challenges such as:
- Misclassified tickets
- SLA breaches
- Inefficient agent allocation
- Inconsistent customer responses

This system addresses these issues by enabling proactive, data-driven support operations.

## System Architecture
- FastAPI backend
- Modular AI decision layers
- Rule + ML hybrid decision engine
- Human-in-the-loop AI assistance
- Analytics and feedback loop

## AI Components
- Intent Classification
- Sentiment Analysis
- Urgency Detection
- SLA Breach Prediction
- AI-Assisted Reply Drafting

## Key Features
- Priority assignment (P1 / P2 / P3)
- SLA risk prediction and escalation
- Skill-based intelligent agent routing
- Explainable AI decisions
- Operational analytics dashboard

## Tech Stack
- Python
- FastAPI
- NLP / ML
- PostgreSQL
- Docker

## Deployment
The system is containerized using Docker and supports local, staging, and production-like environments.
